## 1. 项目概述

### 1.1 背景
NotebookLM 的学习能力很强，但存在以下痛点：
- 只能逐个添加链接
- 最多50个视频链接限制
- 手动操作繁琐，无法自动化批量处理

### 1.2 目标
创建一个自动化学习系统，能够：
- 批量分析 YouTube 博主的所有视频
- 自动提取字幕并生成结构化笔记
- 自动识别名词并提供科普解释
- 每日增量分析，无需人工干预
- 可选同步到 NotebookLM

### 1.3 核心指标
- 支持最多 10 个博主同时监控
- 每个博主每天分析 5 条视频
- 自动生成标签、摘要、科普内容

## 4. 功能模块

### 4.1 飞书交互模块
**指令集：**
- `@Lip 添加博主 <URL>` — 添加新博主监控
- `@Lip 状态` — 查看所有博主进度
- `@Lip 删除博主 <博主名>` — 停止监控并清理数据
- `@Lip 同步 <博主名> 到NotebookLM` — 手动触发同步
- `@Lip 科普词典` — 查看所有名词解释汇总

### 4.2 定时分析引擎
**每日任务 (cron: 0 2 * * *)：**
1. 读取 config.json 获取所有活跃博主
2. 对每个博主：
   - 获取频道最新视频列表
   - 筛选未分析的视频（按最新排序）
   - 取前5条进行处理
3. 对每条视频：
   - 提取字幕（yt-dlp）
   - AI生成摘要
   - AI识别并解释名词
   - AI生成标签
   - 保存到本地 Markdown
   - 更新统计信息
4. 发送飞书日报（处理结果汇总）

### 4.3 Web 面板（集成到 Gallagher）
**页面结构：**
- **仪表板**：博主卡片（进度条）、今日分析数量
- **笔记列表**：搜索、标签筛选、排序
- **科普词典**：名词A-Z索引、搜索
- **设置**：添加/删除博主、NotebookLM同步设置

## 6. API 设计

### 6.1 YouTube 处理
```typescript
// 获取频道视频列表
getChannelVideos(channelUrl: string): VideoInfo[]

// 提取视频字幕
extractTranscript(videoId: string): string

// 分析视频内容
analyzeVideo(transcript: string): AnalysisResult
```

### 6.2 AI 分析 Prompt

**摘要生成：**
```
请为以下YouTube视频字幕生成结构化摘要：
1. 3-5条核心要点
2. 每个要点简洁明了

字幕内容：
{transcript}
```

**标签生成：**
```
请分析以下内容，提取3-5个关键词标签：
- 标签应该反映内容主题
- 格式：#标签名

内容：
{transcript}
```

**名词识别与科普：**
```
请识别以下内容中提到的所有人名、地名、历史事件、专业术语、概念等专有名词，并为每个名词提供简要解释（50-100字）。

格式要求：
- 名词：解释内容

内容：
{transcript}
```

## 7. 错误处理

### 7.1 重试策略
- YouTube提取失败：重试3次，间隔5分钟
- AI分析失败：重试2次，记录到错误日志
- 飞书同步失败：发送告警通知，不阻塞主流程

### 7.2 错误类型
- `NETWORK_ERROR`: 网络问题，自动重试
- `NO_TRANSCRIPT`: 视频无字幕，跳过并标记
- `AI_TIMEOUT`: AI超时，稍后重试
- `RATE_LIMIT`: 触发限流，暂停并通知

---

## 8. 安全与限制

- 每日最大分析量：10博主 × 5视频 = 50条
- YouTube请求频率：使用代理池避免IP被封
- AI Token限制：单视频字幕截断至8000字符（如有必要）
- 数据隐私：所有数据本地存储，飞书同步需显式授权
